第30天：撰写阅读说明

Creating an Accessibility Statement  撰写阅读说明

既然你已经采用了这一系列的技巧，现在就该撰写一份阅读说明，详细列出网站上所有的亲和力特色。

谁因此获益？

  1. 老路 从中获益了。因为他十分仰赖键盘来导航，而且如果他能够知道相关讯息的话，就很可能会用到你的accesskey 快速键。很不幸地， Mozilla 不会用任何方法把这些设定念出来或者显示出来给使用者看，所以你得自己列出来。
  2. 许多其他使用者都从中获益了，因为他们能够由此得知，如何把她们自己的软件 用得更出神入化。举例来说， 马大姐 并不知道她可以设定浏览器的预设文字尺寸，直到有人跟她说可以这么做；现在她终于可以更容易地阅读你的网站了（当然，你得先使用相对字体大小）。 Netscape 和 Mozilla 的使用者也许并不知道她们能够开启网站导航列，从你以链结为主的导航协助中获益（“检视”选单，“显示／隐藏”，“网站导航列”，“需要时才显示”。很不幸地，这个功能在最后一刻从 Mozilla 1.0 最终版里被拿掉了，不过 1.1 版时又被加回来了）。
  3. 其他网页设计师从中获益了，因为她们将能够看到且学到你所花时间采用的这些亲和力技巧。

怎么做

在 Radio 或 Manila 中，你可以把阅读说明当作是一则故事（ Radio 或 Manila 工具列上的“ Stories ”）来发表，所以它将有自己的网址。要不然就是直接建立起新的文章，并且在储存后把它的网址记下来；稍后你将会用上。

你第一件该做的事情是列出你所定义过的 accesskey 快速键，因为多数的浏览器既不会把这些快速键念出来，也不会让他们被显示出来。同时也要记得列出阅读说明本身的 accesskey （通常会是 accesskey="0" ，我们马上就会看到了）。

在这之后，你还该列出所有你采用的亲和力特色。如果你只有采用某几个技巧的话，就只提出你真的用上的那几个就好。这会是很有启发性的经验，就跟你在更新履历时一样；当你第一次得到工作时，你将了解到你到底曾经做过了些甚么事。你已经做很多事了。

如果你高兴的话，还可以概略地描述一下为什么要采用这些特色（像是“对于屏幕阅读软件的使用者来说会更好用”、“这是设计给纯文字浏览器的使用者的”、“这是为了无法分辨颜色的使用者所设计的”之类）。请记住，阅读说明可能会被更多残障的使用者所阅读，所以它不但该具备详尽的情报性（给那些直接获益的人），还该具备教育性（给那些拿你的网站来当范例的网页设计师）。或者除了你自己解释之外，你还可以连结到这本书里的任一天的内容，当然也可以就只是连结到本书的首页。

最后，让你的网站里每一页都连结到你的阅读说明；你可以放在全站的导航列，也可以放在每一页的注脚。指定这个链结的标题以及 accesskey="0" ，就像这样：

<a href="accessibility_statement.html" title="本站的亲和力特色" accesskey="0">阅读说明</a>

延伸阅读

  • diveintomark.org 的阅读说明。
  • diveintoaccessibility.org 的阅读说明。你可以把页面的结构、字句及延伸阅读的链结这些当成模版，自由取用。
