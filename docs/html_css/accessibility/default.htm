30 天打造更具亲和力的网站

Dive Into Accessibility

原作者：Movable Type 翻译整理：CodeCoke 时间：2005-1-15

我们将回答两个问题：

第一个问题是：“为什么该让我的网站更有亲和力？”如果你根本就没有网站的话，显然就不会成为本书的读者了。

第二个问题是：“怎么让我的网站更有亲和力？”如果第一个问题的回答说服不了你，那么第二个问题你大概也没兴趣。

阅读 Dive Into Accessibility

目录
所有建议皆根据原本出版的顺序来编排。建议读者们第一次阅读本书时，先按照这个顺序阅读。
按人物不同阅读
按照每种建议的受益者来编排。
按障碍不同阅读
按照每种建议所能解决的障碍来编排；所谓的障碍不止包含了身体上的，也包含了技术上的。
按设计原则不同阅读
按照每种建议所依循的 W3C 原则来编排。这指的是所有的页面都应该要具备可认知 (perceivable) 、好操控 (operable) 、良好导航 (navigable) 、易理解 (understandable) 以及够强健 (robust) 等原则。
依浏览器分类
按照网页浏览器与辅助技术来编排，这些工具将会因这些建议而受益。
按出版工具不同阅读
按照出版工具可采用的建议来编排；每一个建议都会针对适用的出版工具提出修正步骤。
