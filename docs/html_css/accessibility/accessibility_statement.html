打造网站亲和力教程说明

statement 阅读说明

以下是深入亲和力的正式声明。如果你有任何疑问或建议的话，欢迎来信 feedback@diveintoaccessibility.org.

快捷键

大部分浏览器都支持用网页上定义的按键来跳到特定的链结。在 Windows 上，你可以按下 ALT + 某个快捷键；在 Macintosh 上，你可以按下 Control + 某个快捷键。

这里的页面都定义了下列的快捷键：

  • 快捷键 h - cob首页Alt + h (windows) 然后按enter键即可返回首页。
  • 快捷键 t - tech首页Alt + t (windows) 然后按enter键即可返回CobTech首页
  • 快捷键 a - Accessibility首页Alt + a (windows) 然后按enter键即可返回本章首页
  • 快捷键 3 - Accessibility目录说明如上
  • 快捷键 S - 站内搜索Alt + s (windows) ，即可将光标定位到搜索输入框。
  • 快捷键 8 - 版权声明
  • 快捷键 C - 回馈
  • 快捷键 0 - 阅读说明

遵循标准

  • 本站所有的页面都通过了 Bobby AAA 认证，遵循所有的 Bobby 指导原则。这一直都是主观判断下的结果；许多亲和力要素可以被测得，但是也有很多没办法。我已经回顾了所有的指导原则，相信所有的页面都确实遵照了。
  • 本站所有的页面都通过了 WCAG AAA 认证，遵循W3C Web Content Accessibility Guidelines 的所有第一优先、第二优先和第三优先指导原则。同样地，这也是主观判断下的结果；许多指导原则都刻意模糊而无法被自动测试。我也回顾了所有的指导原则，相信所有的页面都确实遵照了。
  • 这个网站上所有的页面都通过了 Section 508 认证，遵循所有的美国联邦政府 Section 508 指导原则。再一次地，这还是主观判断下的结果。我已经回顾了所有的指导原则，相信所有的页面都确实遵照了。
  • 本站所有的页面都通过了 XHTML 1.0 Strict 认证。这并非主观判断；有一个程式能够 100% 精确地判断某个页面是否为有效的 XHTML 。举例来说，检查主页面是否为有效的 XHTML 。
  • 本站所有的页面都使用了结构化语意标记。 H2 标签都用于主标题， H3 标签都用于副标题。举例来说，在这一页里， JAWS 的使用者可以按下 ALT+INSERT+3 来跳到阅读说明的下一个小节。

导航协助

  • 所有的页面都有 rel=previous, next, up, 和 home 链结，来协助纯文字浏览器导航。 Netscape 6 和 Mozilla 的使用者也可以从检视选单、显示／隐藏，网站导航列、只在需要的时候显示（或永远显示）来启用这个进阶功能。
  • 所有的技巧都可以用多种不同的方法交叉参考。你可以依人物、依障碍、依设计原则、依协助科技或依出版工具来浏览这些技巧。
  • 主页面以及所有的汇整页面都有一个搜索框（快捷键 4 ）。进阶搜索选项也可以在进阶搜索页面里使用。

链结

  • 许多的链结都以 title 属性来更详细地描述链结，否则就是因为链结的文字已经完全能够描述连结的目标（像是文章的标题）了。
  • 链结即使离开了情境依旧有意义。

图片

  1. 本站所有的内容图片都包括了描述性的 ALT 属性。单纯的装适用图片则包含了空的 ALT 属性。
  2. 复杂图片则会包含了 LONGDESC 属性或者是内嵌了描述，为非视觉的读者描述每一张图片的含意。

视觉设计

  • 本站使用了样式表串联来处理视觉排版。
  • 本站仅使用相对字体大小，与使用者在视觉性浏览器指定的“字体大小”相容。
  • 如果你的浏览器或浏览装置完全不支援样式表的话，每一页的内容仍旧可阅读。

亲和力参考文献

  1. W3 亲和力指导原则解释了每一个指导原则背后的理由。
  2. W3 亲和力技术解释了如何采用每一个指导原则。
  3. W3 亲和力核对表可以作为忙碌开发者的亲和力指引。
  4. 美国联邦政府 Section 508 亲和力指导原则。

亲和力软件

  1. JAWS 是 Windows 上的屏幕阅读软件，这里提供了可以下载的限时展示版。
  2. Home Page Reader 是 Windows 上的屏幕阅读软件，这里提供了可以下载的展示版。
  3. Lynx 是一个免费的纯文字浏览器，搭配可覆新的布拉耶点字显示器后就可以给盲人使用。
  4. Links 是一个给频宽不足的视觉性使用者用的纯文字浏览器。
  5. Opera 是一个配备许多亲和力功能的视觉性浏览器，包括有文字缩放、自订样式表、图片标定等多样功能。有提供免费下载的版本。与 Windows 、 Macintosh 、 Linux 和许多其他的作业系统相容。

亲和力服务

  1. Bobby 是一个分析网页是否遵照亲和力指导原则的免费服务。也提供全功能的商业版本。
  2. HTML Validator 是一个用来检查网页是否吻合已公开的 HTML 标准的免费服务。
  3. Web Page Backward Compatibility Viewer 这个工具可以用来检视当你的网页，看看若没有各种流行的浏览器功能时，它会变成怎样。
  4. Lynx Viewer 是一个让你检视网页在 Lynx 里看起来会像甚么的免费服务。

相关资源

  1. WebAIM 是一个专注于改善线上学习材料亲和力的非营利组织。
  2. Designing More Usable Web Sites 列出了大量额外的资源。

我建议的亲和力书籍

  • Joe Clark: Building Accessible Websites. 我是这本书的技术编辑，这真的是本博而不滥的好书。
  • Jim Thatcher and others: Constructing Accessible Web Sites. 比 Joe 的书精简一点，但是它所囊括的主题深度更为过之。这本书还提供了许多的撷取画面，展示出各种屏幕阅读软件及浏览器在各种不同的标签及标记上所显示的效果；同时还有一个章节描述着跟亲和力需求有关的法律声明。
